## 二、审查基石：输入对象与任务定义

### 3. 审查输入体系分类视图
明确了系统执行审查时的两类基石输入，在前端工作台表现为四类文件输入口径：
1. **待审文件 (TARGET_DOC)**：如施工组织设计、一般方案、及具有三级分类结构的专项施工方案。
2. **审查依据 (REVIEW_BASIS)**：系统预设的法律法规与标准规范。
3. **项目资料 (PROJECT_CONTEXT)**：用户补充的项目环境上下文，如合同、图纸、勘察报告、招标文件等。
4. **补充参考 (REFERENCE_DOC)**：用户补充的其他非标准性参考文档。

### 5. 审查任务编译层 (Task Compiler / Review Brief)
前端表单的输入不会直接驱动底层大模型，而是会由系统拦截并进行“编译”，统一打包成任务书（Review Brief）。
该任务书旨在消除输入侧的不确定性，形成标准化的契约，随后同时分发给底层的 008 与 Hermes 引擎进行双路审查。

**审查任务书（Review Brief）核心结构包含：**
- `review_id`: 任务全局唯一标识
- `review_object_type`: 审查对象类型（如：危大工程-基坑工程）
- `target_files`: 待审文件列表
- `basis_files`: 审查依据列表
- `project_context_files`: 项目资料列表
- `focus_pack`: 审查重点集合（预设重点 + 附加要求）
- `review_policy`: 审查口径（如：保守 / 标准 / 严格）
- `report_type`: 报告类型（如：简版结论 / 详细报告）

## 三、双路审查与结果融合架构

当前系统采用 **008 + Hermes 双路协同架构**。前端对用户不暴露双引擎差异，而是统一展现为一致的工作台，后台由两套流并行推进。

### 6. 008 结构化审查事实底稿链路
- **职责**：作为主审查链，负责重体力劳动与事实提取。
- **机制**：读取任务书，进行长文本解析、结构化信息抽取。
- **产出**：输出 `008_fact_packet.json`（包含结构化的 finding、evidence status、basis refs 等）。
- **定位**：提供稳定、可控、可追溯的结构化事实底稿，作为下游引擎进一步研判的基石。

### 7. Hermes 第二路审查链路
- **职责**：作为旁路增强，基于同一份任务书独立进行重点导向审查与第二意见审查。
- **机制**：与 008 并行或基于 008 事实底稿进行上层逻辑推理。
- **产出**：输出 `hermes_review_packet.json`。
- **定位**：专注于高阶逻辑判断和重点问题深挖。

### 8. Hermes 融合与最终报告生成层
- **职责**：Hermes 在完成自身审查后，统一接收并融合 `review_brief.json`、`008_fact_packet.json` 和 `hermes_review_packet.json`。
- **机制**：基于设定的“审查口径”与“报告类型”，对两路审查结果进行交叉比对与去重，形成最终结论。
- **产出**：生成最终对外报告成品（如 `final_report.md` 或导出 Word/PDF）。
- **定位**：系统对前端只暴露一份最终报告，既包含可读性强的总结，也保留了底稿的可追溯性证据。

### 11. 核心组件与状态联动机制
- **级联定位器与卡槽联动**：三级节点选定后，自动刷新下方的预设规范、预设重点，并展示分类别的文件上传卡槽。文件分类包括待审文件、审查依据、项目资料等。
- **统一编译与下发**：点击提交时，前端收集所有的文件标识（file_id）与要求配置，请求一次性创建任务，由后端编译器统一封装为审查任务书。
- **状态感知工作台**：针对长耗时任务，通过 SSE 订阅阶段进度（如编译、双路审查、融合报告）。交付结果为一份由 Hermes 融合的终版静态审查报告（附带可信度说明与补充资料建议）。

## 五、反馈沉淀与后续演进

### 12. 轻反馈闭环设计
为保证系统的低摩擦迭代，不在用户侧施加过重的“人工逐条复核队列”认知，最终结果页仅保留极简的反馈操作。最终收敛为以下 4 个动作：
- **👍 有帮助**：确认该审查项的价值。
- **👎 问题不准**：标记结论推断错误或过度敏感。
- **📝 有遗漏**：快速标记当前审查未覆盖到的盲点。
- **💾 存为模板**（可选）：将当前修改后的“附加要求”一键沉淀。

### 13. 模板沉淀与后续整合方向
轻反馈的目标不是为了复杂的审核运营，而是为了提供最小真实的闭环数据。这些数据将直接用于：
- **资产化**：将高频的优质附加要求沉淀为租户级别的预设审查重点模板。
- **引擎调优**：为后续 008 底稿规则调优与 Hermes 大模型 Prompt 优化提供标注数据。
