# review-rule-extractor Skill 设计文档

## 1. 概述

### 1.1 目标

创建一个 Claude Code Skill，根据用户指定的方案类型和审查依据法规/标准，从工作区 `review-rules/` 目录中的 .md 文件动态抽取审查规则，输出符合 Hermes 审查系统 `ReviewRequest.review_rules[]` 格式的 JSON 数组。

### 1.2 核心价值

- **自动化规则抽取**: 替代人工从法规文件中逐条提取审查规则的工作
- **标准化输出**: 输出格式直接可用于 Hermes 审查引擎
- **依据溯源**: 每条规则附带法规来源（标准名称 + 条文编号）
- **动态适配**: 支持任意方案类型和法规组合

### 1.3 Skill 位置

```
/Users/lucas/repos/agent-skills/review-rule-extractor/
```

通过软链接接入 Claude Code、Codex 等工具。

## 2. 输入输出规格

### 2.1 输入

用户通过自然语言描述需求，skill 从中解析以下参数:

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `scheme_type` | string | 是 | 方案类型，如 "基坑工程"、"停电施工方案" |
| `regulations` | string[] | 是 | 审查依据的法规/标准名称列表 |
| `review_items` | string[] | 否 | 要启用的审查模块，默认全部5个 |

**输入示例**:
```
帮我抽取基坑工程的审查规则，依据 JGJ 120-2012 和 GB 50497-2019
```

### 2.2 输出

`review_rules[]` JSON 数组，每个元素结构:

```json
{
  "rule_id": "JGJ120-3.1.5",
  "review_item_code": "content_compliance",
  "rule_name": "高压电气设备交接试验",
  "rule_description": "高压电气设备、布线系统及继电保护系统必须交接试验合格（依据：《建筑基坑支护技术规程》JGJ 120-2012 §3.1.5 强制性条文）",
  "enabled": true,
  "based_on_review_basis": false
}
```

**字段说明**:

| 字段 | 类型 | 说明 |
|------|------|------|
| `rule_id` | string | 使用法规条文编号，格式为 `{标准编号}-{条文号}`，如 `JGJ120-3.1.5` |
| `review_item_code` | string | 审查模块编码，5选1 |
| `rule_name` | string | 规则简短标题，10字以内 |
| `rule_description` | string | 规则描述，必须包含依据来源 |
| `enabled` | boolean | 固定为 `true` |
| `based_on_review_basis` | boolean | 固定为 `false`（规则来自系统预置，非用户传入的审查依据附件） |

## 3. 五大审查模块定义

| code | 名称 | 定义 | 典型规则特征 |
|------|------|------|-------------|
| `structure_integrity` | 结构完整性 | 方案是否包含本类型工程所要求的必备内容、章节和格式要件 | 涉及"必备章节"、"目录"、"附件"、"应包含" |
| `basis_accuracy` | 编制依据准确性 | 引用的法规、标准、规范是否准确、适用、现行有效 | 涉及"引用标准"、"规范编号"、"现行有效"、"编制依据" |
| `consistency_check` | 一致性复核 | 方案内部前后文、正文与计算书之间的关键参数是否一致 | 涉及"参数一致"、"前后文对照"、"交叉验证" |
| `content_compliance` | 方案内容合规性 | 施工工艺、安全措施、质量控制等内容是否满足规则和依据要求 | 涉及"安全措施"、"施工工艺"、"质量控制"、"应急"、"验收" |
| `calculation_review` | 计算书审核 | 计算书的参数、模型、过程、结论是否完整正确 | 涉及"计算"、"验算"、"承载力"、"稳定性"、"公式" |

### 3.1 模块分类指导原则

LLM 在分类时应遵循以下优先级:

1. **内容优先于关键词**: 根据条文的实际业务含义判断，而非仅匹配关键词
2. **安全措施类条文归入 content_compliance**: 大部分安全操作规程条文属于内容合规性
3. **结构要求类条文归入 structure_integrity**: 要求方案"应包含"某内容的条文属于结构完整性
4. **计算相关条文归入 calculation_review**: 涉及计算方法、参数取值的条文
5. **引用标准相关条文归入 basis_accuracy**: 涉及标准引用、版本核验的条文
6. **一致性相关条文归入 consistency_check**: 涉及跨章节对照、参数核验的条文

## 4. 内部处理流程

```
用户自然语言输入
  ↓
Step 1: 意图解析
  - 提取 scheme_type
  - 提取 regulations 列表
  - 确认 review_items（默认全部）
  ↓
Step 2: 文件定位
  - 在 review-rules/ 目录下搜索匹配的 .md 文件
  - 支持模糊匹配（标准编号、标准名称）
  - 找不到时提示用户确认
  ↓
Step 3: 结构化解析
  - 读取 .md 文件内容
  - 识别条文编号格式（见4.1）
  - 提取每个条文的完整文本
  - 识别强制性条文标记
  - 输出: 条文清单
  ↓
Step 4: LLM 分类 + 规则生成
  - 将条文清单交给 LLM
  - 判断每条条文属于哪个审查模块
  - 生成 rule_name 和 rule_description
  - 强制性条文标注"强制性条文"
  ↓
Step 5: 输出组装
  - 生成 rule_id = "{标准编号}-{条文号}"
  - 按审查模块分组排序
  - 输出 review_rules[] JSON
```

### 4.1 条文识别模式

skill 识别以下 .md 文件中的条文格式:

| 模式 | 示例 | 正则 |
|------|------|------|
| 数字编号列表 | `- 3.1.5 施工前应...` | `/^\s*-\s+(\d+\.\d+\.\d+)\s+(.+)/` |
| 行首数字编号 | `3.1.5 施工前应...` | `/^(\d+\.\d+\.\d+)\s+(.+)/` |
| §前缀 | `§5.2.4 现场勘察应...` | `/§(\d+\.\d+\.\d+)\s+(.+)/` |
| 中文格式 | `第6.2.2条 所有电源...` | `/第(\d+\.\d+\.\d+)条\s+(.+)/` |
| 强制性条文标记 | `> ⚠️ **强制性条文**` | 识别 callout 块 |

### 4.2 强制性条文识别

.md 文件中通常使用以下标记:

```markdown
> ⚠️ **强制性条文**
> - 3.1.5 高压电气设备必须交接试验合格。
```

或:

```markdown
**强制性条文** 3.1.5 高压电气设备必须交接试验合格。
```

skill 识别这些标记，并在 rule_description 中注明"强制性条文"。

## 6. 边界情况处理

| 场景 | 处理方式 |
|------|---------|
| 法规文件在 review-rules/ 中找不到 | 提示用户确认文件名，列出模糊匹配结果 |
| .md 文件格式不规范，无法识别条文 | 将整个文件内容交给 LLM 处理，标注"条文识别置信度低" |
| 一条条文可能属于多个模块 | 允许多归属，生成多条规则（rule_id 相同，review_item_code 不同） |
| 强制性条文 | 在 rule_description 中注明"强制性条文" |
| 用户未指定方案类型 | 提示用户补充，或根据法规内容推断 |
| 法规文件内容过长 | 按章节分批处理，合并结果 |
| 条文编号格式不标准 | 尽量提取，无法提取时使用文件名+序号作为 rule_id |

## 7. 核心约束

1. **rule_id 使用法规条文编号**: 格式为 `{标准编号}-{条文号}`，如 `JGJ120-3.1.5`
2. **rule_description 必须包含依据来源**: 格式为 `规则描述（依据：《标准名称》标准编号 §x.X.X）`
3. **不编造条文**: 只从 .md 文件中实际存在的条文生成规则
4. **不判断法规有效性**: 不输出"已废止"等结论，只如实引用
5. **输出仅为 review_rules[]**: 不生成完整 ReviewRequest
6. **强制性条文必须标注**: 在 rule_description 中注明"强制性条文"

| 编号 | 验收项 | 目标值 |
|------|--------|--------|
| A-01 | 法规文件定位准确率 | 100%（文件存在时必须找到） |
| A-02 | 条文识别覆盖率 | ≥90%（标准格式的条文必须识别） |
| A-03 | 模块分类准确率 | ≥85%（人工抽检） |
| A-04 | rule_id 格式正确率 | 100% |
| A-05 | rule_description 包含依据来源 | 100% |
| A-06 | 强制性条文标注率 | 100% |
| A-07 | 不编造条文 | 100%（输出条文必须在源文件中存在） |
