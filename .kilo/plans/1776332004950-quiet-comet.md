# Hermes 架构研究：证据溯源 (Provenance) 与关联 (Relationship) 深度重构计划

## 1. 左侧：方案原文（设计目标与架构）

### 1.1 现状痛点
- **弱类型字典绑定**：事实与证据通过字符串 Key 隐式关联，易在清洗/升级中静默断裂。
- **全量对象冗余复制**：EvidenceSpan 在提取→写入→输出链路被整体拷贝，导致不一致风险。
- **防幻觉门禁缺失**：SupportPacket 直接把 facts 传给 Hermes 内核，无法在工程层阻止 LLM 臆造/篡改证据文本。

### 1.2 核心思想
将「内嵌式证据」升级为「基于注册表的指针式溯源」：
- **全局证据注册表（Provenance Registry）**：对每份 EvidenceSpan 生成唯一 span_id 并封存。
- **事实标注（Annotation as Pointers）**：事实携带 provenance_ids 列表，指向注册表中的证据节点。
- **关系图谱（Fact Relationships）**：用 typed edges 明确定性关系（contradicts/supports/derived_from），避免大模型在字符串中描述关系。

### 1.3 架构要点
- 仅在控制面实现，不侵入 external/hermes-agent。
- 保持现有数据模型兼容，通过 wrapper/adapter 在边界注入新结构。
- 在输出端（Assembler）实施硬校验：仅允许引用注册表中存在的 span_id。

## 2. 右侧：审查问题定位与原文高亮

| 问题类别 | 审查问题原文 | 对应原文位置（链接） | 标记要点 |
|---|---|---|---|
| 证据引用缺失 | 未提供有效 provenance_ids，Fact 仅含 value | `models.py:EvidenceSpan` / `support_packet_builder.py:SupportPacket` | **🔴 红色**：缺少注册表引用，视为幻觉 |
| 冗余复制风险 | EvidenceSpan 被多处 `model_dump()` 传递 | `support_packet_builder.py:build_packet` → `_load_basis_fulltext_context` | **🟡 橙色**：对象级拷贝，建议改用引用 |
| 防幻觉门禁缺口 | Assembler 未校验 LLM 返回的 span_id 合法性 | `support_packet_builder.py:_collect_expert_review_points` / `assembler.py`（未展示） | **🔴 红色**：需增加 span_id 校验逻辑 |
| 关系未结构化 | 关系依赖大模型文字描述，未显式建模 | 当前设计未包含 FactRelationship 实体 | **🟡 橙色**：建议引入并显式存储 |
| 注释干扰生产 | 方案文档含实现注释混入正式设计 | 本计划文档（本文件） | **🟢 绿色**：仅文档层面，代码不受影响 |

### 跳转与高亮策略（设计时约定）
- 为每个审查问题生成锚点链接（例如 `#evidence-span-provenance`），点击后滚动至原文位置。
- 在原文对应代码块/段落应用内联高亮（CSS 类 `.highlight-red/.highlight-orange/.highlight-green`），审查视图中展示彩色背景与行号标记。
- 在 Assembler 输出中，将 span_id 与注册表做匹配，失败则置灰并标注 `[幻觉风险]`。

## 3. 实施路线图（与左侧对应）

### 阶段 A：模型增强（左）
- 在 `models.py`/`schema.py` 新增 `span_id: str` 字段（EvidenceSpan）。
- 新增 `AnnotatedFact` 与 `FactRelationship` 实体，明确 provenance_ids 与关系类型。
- 不破坏现有接口，通过 adapter 在边界做类型转换。

### 阶段 B：边界收紧（中）
- 修改 `support_packet_builder.py`：在 `build_packet` 中构造 `provenance_registry`，并为每个 Fact 生成稳定 `span_id`。
- 实现 `_collect_expert_review_points` 的校验开关：仅当 span_id 存在于 registry 时才收集为专家点。

### 阶段 C：输出端硬门（右）
- 在 `Assembler` / `FinalReportRenderer` 增加校验层：遍历输出的 Issue，若引用证据则校验 span_id；失败则降级为 `evidence_gap` 并高亮标记。
- 提供审查 UI 组件：左侧方案原文 + 右侧问题列表，点击跳转与高亮。
