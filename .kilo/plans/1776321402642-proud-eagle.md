# 方案类型到子Agent启动机制核查报告及修复计划

## 1. 机制现状分析
前端选择“方案类型”（`documentType`）和启用的审查模块（`enabled_modules`）后，后端的处理链路如下：

1. **请求接收与转换**: `apps/api/src/routes/tasks.py` 接收到请求，提取 `documentType` 以及 `reviewIntent.enabled_modules`。
2. **模块到Agent的映射**: 后端通过 `module_template_ids(enabled_modules)`（定义在 `module_bindings.py`）将启用的模块名称（如 `structure_completeness`）转换为对应绑定的**所有**模板 ID 列表（如 `['structure_completeness_reviewer', 'construction_org_structure_reviewer']`）。此时没有进行方案类型的过滤，将该列表存入 `plan.hermesInput.enabledAgents`。
3. **Agent注册表选择**: `HermesController` 调用 `HermesTemplateRegistry.select_templates`，根据 `brief.review_object_type`（方案类型）和 `hermes_input.enabledAgents` 选择要启动的子 Agent。

## 2. 发现的严重漏洞 (Bug)
在 `apps/api/src/review/hermes/template_registry.py` 的 `select_templates` 方法中，存在违反“**`supported_document_types` 是 reviewer 隔离的唯一机制**”规则的代码逻辑：

```python
            if enabled:
                if template.id in enabled:
                    score += 100
                    reasons.append('explicitly_enabled')
                else:
                    continue
            
            # 【漏洞所在】
            if not template.supported_document_types or str(brief.review_object_type) in template.supported_document_types:
                score += 5
                reasons.append('document_type_match')
            
            # ... 省略其他加分项 ...

            if score > 0:
                matches.append(AgentTemplateMatch(template=template, score=score, reasons=reasons))
```

**问题说明**：
当某个模块被启用时，该模块下绑定的**所有**审查器（包括专属于 `construction_org` 施组的审查器）都会被加入 `enabledAgents` 列表。
在匹配时，因为它们在 `enabledAgents` 中，直接获得了 `100` 分。
接着判断 `supported_document_types`，如果不匹配当前的 `review_object_type`，只是**拿不到这 5 分的加分**，但最后判断 `if score > 0` 依然成立（100 > 0），导致**不支持当前方案类型的专属审查器依然被选中并执行**。这会造成审查结果被其他文档类型的规则污染（例如配网工程被施组专属规则审查）。

## 3. 修复计划 (Proposed Fix)
在 `apps/api/src/review/hermes/template_registry.py` 中，将 `supported_document_types` 的校验改为**硬性门禁 (Hard Gate)**。

需要修改的逻辑如下：
```python
        for template in self.load_templates():
            if template.id in disabled:
                continue
                
            # 【新增：硬性过滤】如果不满足支持的文档类型，直接跳过，无视是否在 enabled 列表中
            if template.supported_document_types and str(brief.review_object_type) not in template.supported_document_types:
                continue

            score = 0
            reasons: list[str] = []
            if enabled:
                if template.id in enabled:
                    score += 100
                    reasons.append('explicitly_enabled')
                else:
                    continue
            
            # 走到这里的都已经 match 了 document_type
            if not template.supported_document_types or str(brief.review_object_type) in template.supported_document_types:
                score += 5
                reasons.append('document_type_match')
```
