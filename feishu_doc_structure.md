### 7. 核心组件库与联动机理 (Component State Management)

**① 级联定位器模块 (Cascader & Slot Mapper)**
- **交互**：用户完成 一级 -> 二级 -> 三级 的选择。
- **状态联动**：一旦选中末级节点（如“基坑工程”），触发防抖 API 请求，动态从后台拉取该类型工程关联的「预设规范库」与「预设重点项」以刷新下方的勾选列表。

**② 结构化文件上传槽 (Slot-based Uploader)**
- **交互**：页面渲染明确的上传卡槽（主文件槽为必选大卡片，辅助依据为标签型次级槽）。
- **数据流承载**：上传接口带参，将文件类型强行与业务类型挂钩（e.g., `file_role=TARGET_DOC`, `file_role=AUX_CONTRACT`），免除大模型在后台做多余的分类与猜测成本。

**③ 提示词资产管理模块 (Prompt Template Manager)**
- **交互**：带历史记录下拉和保存按键的多行文本框。
- **关联接口**：
  - `GET /api/v1/prompt-templates` (拉取租户下的常用审查侧重点)
  - `POST /api/v1/prompt-templates` (保存当前规则模板)

**④ 静态产出与状态工作台 (Static Report Delivery)**
- **进度展示**：针对大模型流式切片审查长耗时场景，配合 WebSocket / SSE 机制展示阶段性处理状态（如“图纸解析中...” -> “标准比对中...”）。
- **结果呈现**：交付包含整体分析得分、高危拦截项汇总与风险清单表格的静态审查简报。
- **动作闭环**：提供标准化的【一键导出 PDF/Word】操作入口，完成审查数据出表流转。
